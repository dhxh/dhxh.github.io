Linux应急响应

发表于2023-03-17|更新于2023-03-17|网络安全

一、账户

  • 用户信息文件
    • /etc/passwd
    • /etc/shadow
    • /etc/group
  • who 查看当前登录用户
  • w 查看系统信息，某一刻用户行为
  • uptime 查看登录多久、多少用户、负载状态

1、是否为弱口令

2、查询是否有可疑账户

  • cat /etc/passwd

3、 查询特权用户

  • awk -F: '$3==0{print $1}' /etc/passwd

4、查询可疑远程登录的账号

  • awk '/\$1|\$6/{print $1}' /etc/shadow

5、查看sudo权限的用户

  • cat /etc/sudoers | grep -v "^#\|^$" | grep "ALL=(ALL)"

6、公私钥免密登录

  • 查看/root/.ssh/authorized_keys是否修改。

7、日志

  • 日志默认位置 /var/log
  • 日志配置 /etc/rsyslog.conf
日志文件 说明
/var/log/cron 记录了系统定时任务相关的日志
/var/log/cups 记录了打印信息的日志
/var/log/dmesg 记录了系统开机时内核自检信息。
/var/log/message 记录了系统重要信息的日志。
/var/log/btmp 记录了错误登录日志
/var/log/lastlog 记录了所有用户最后一次登录时间的日志
/var/log/wtmp 永久记录所有用户的登录、注销，同时记录系统启动、重启、关机事件。
/var/log/utmp 记录了当前已经登录的用户信息
/var/log/secure 记录验证和授权方面的信息，涉及账号和密码的程序都会记录

二、历史命令

  • cat /root/.bash_history
  • cat /home/$USER/.bash_history

三、端口和进程

1、检查是否有异常连接的端口

  • netstat -antlp
  • ls -l /proc/$PID/exe
  • lsof -i:port

2、检查异常进程

  • 注意事项
    • 有没有命名奇怪的进程
    • 注意cpu占用
    • kill -9 PID后是否重启复活
  • 检查方法
    • ps -aux
    • lsof -p PID
    • 工具unhide

四、开机启动、计划任务、服务

1、开机启动项

  • 开机启动目录
    • /etc/init.d/
    • /etc/rc.local
    • /etc/rc.d/rc[0~6].d

2、计划任务

  • 计划任务目录
    • /var/spool/cron/
    • /var/spool/anacron
    • /etc/crontab
    • /etc/cron.d
    • /etc/cron.daily
    • /etc/cron.hourly
    • /etc/cron.monthly
    • /etc/cron.weekly
    • /etc/anacrontab

3、服务

  • 服务自启动

    • chkconfig

      • chkconfig -add 增加服务
      • chkconfig -del 删除服务

    • systemctl

      • systemctl list-unit-file 查看服务开机启动状态
      • systemctl enable 开启服务开机启动
      • systemctl disable 关闭服务开机启动

    • 加入开机启动目录

五、系统信息

1、可疑目录和文件

  • 检查敏感目录
    • /tmp
    • /var/tmp
    • /usr/tmp
  • find 查找最近修改过的文件
    • find ./ -name '*' -mmin -60 查找60分钟内修改的文件
    • find ./ -name '*' -mtime 0 查找24小时内修改的文件
    • find ./ -ctime -2 查找48小时内修改的文件
  • find 指定时间段内修改的文件
    • find ./ -newer file1 ! -newer file2 列出比file新比file2旧的文件
  • find 查找特殊权限的文件
    • find / -perm -4000
    • find / -perm -u=s -type f 2>/dev/null
  • start 确认可疑文件时间

2、系统环境变量

  • echo $PATH

3、命令替换排查

  • 执行一些命令参数，检查是否被替换
  • 检查md5sum和文件大小
  • ls -lat 按时间排序查看是否有修改的痕迹
